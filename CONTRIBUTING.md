# Contributing to Tintero Theme Template

Thank you for your interest in contributing! This document provides guidelines for contributing to Tintero themes.

## 🐛 Reporting Issues

If you discover a missing CSS variable or encounter a problem:

1. Check if the issue has already been reported
2. Provide a clear and detailed description
3. Include screenshots when applicable
4. Specify your Tintero version

## 💡 Suggesting Improvements

We welcome suggestions for:

- New CSS variables that should be documented
- Better examples or explanations
- Additional theme examples
- Documentation improvements

## 🔧 Theme Creation

### Creating Your Theme

1. **Fork this repository** or create a new one, ensuring your root directory contains:
   - `theme.css` - Your theme's CSS variables and styles
   - `theme.json` - Theme metadata and configuration
   - `preview.png` - A preview image of your theme
   - `README.md` - Documentation for your theme

2. **Test locally**: You can load your theme branch directly into Tintero for testing.

3. **Share with the community** (optional):
   - Fork the [Releases-Tintero](https://github.com/fnadalrod/Releases-Tintero) repository
   - Add your theme to `themes/theme-index.json`
   - Submit a pull request for review and approval

## 📝 Documentation Guidelines

- Use clear, concise language
- Include examples where helpful
- Keep formatting consistent
- Update table of contents if needed

## 🧪 Testing

Before submitting your contribution:

1. Validate your CSS syntax
2. Test your theme in Tintero
3. Ensure the preview image displays correctly
4. Verify that all documentation links work properly

## 📄 License

You may use any license you prefer, provided that your repository remains public and accessible to the Tintero app. 

## ❓ Questions

If you have questions, feel free to:

- Open an issue for discussion
- Check existing documentation
- Ask in the Tintero community

Thank you for helping improve Tintero! 🎨
